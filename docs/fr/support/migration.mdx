# Migration de Teta à Theta

Si vous utilisez actuellement Teta et que vous souhaitez migrer vos données vers Theta, nous proposons un service de migration de données dédié pour rendre la transition aussi fluide que possible. Veuillez noter que si Theta et Teta partagent certaines similitudes, Theta est conçu dans un but différent et a une portée différente.

## Commencer

Pour demander une migration de données de Teta vers Theta, veuillez visiter https://buildwiththeta.com/migration et remplir le formulaire avec les informations suivantes :

- nom et prénom
- E-mail
- ID de projet (séparés par des virgules)

Pour trouver vos ID de projet, ouvrez simplement un projet dans Teta et copiez l'ID numérique de l'URL, qui devrait ressembler à ceci : .../editor/$projectId/...

❗ Veuillez noter que la migration des données peut ne pas être possible pour tous les projets, car Theta utilise une structure de base de données différente et certains nœuds peuvent ne plus être utilisés dans le nouveau produit. Nous examinerons chaque demande de migration au cas par cas pour déterminer si cela est possible.

## Ce qui peut être migré

Les données suivantes peuvent être migrées de Teta vers Theta :

- Projets
- Canaux pour chaque projet
- Pages et composants pour chaque canal
- Nœuds pour chaque page ou composant

## Motifs de rejet

Veuillez noter que nous nous réservons le droit de rejeter les demandes de migration pour les raisons suivantes :

- Le nom et l'e-mail fournis ne correspondent pas aux données enregistrées dans Teta
- ID de projet incorrects fournis
- Projet jugé inadapté ou inadapté à la migration
- Contenu inapproprié dans les projets
- Infaisabilité technique de la migration pour des raisons spécifiques

Merci de considérer Theta comme votre nouvelle plate-forme, et nous sommes impatients de vous aider dans le processus de migration des données.
